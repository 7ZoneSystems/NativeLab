Talk to your documents

Load one PDF or a whole folder of them. NativeLab chunks, indexes, and feeds relevant sections into the model's context window automatically. You ask questions naturally — it finds the right passages.

Multi-PDF mode summarizes each document, then synthesizes a final answer across all of them. Ideal for literature reviews, legal discovery, or comparing research papers.

PDF ingestion Multi-document synthesis Context-aware chunking

Reasoning into Code

Assign one model as your reasoning engine and another as your coding engine. When you submit a task, the reasoning model thinks through the intent and architecture, then passes a concise brief to the coding model to generate the implementation.

The result: better code, with less back-and-forth. You get the full chain of thought in the chat so nothing is hidden.

Two-model pipeline Visible reasoning trace Code copy buttons

Parallel Model Roles

Load multiple GGUF models at once, each assigned a role: general chat, reasoning, summarization, coding, or secondary. Switch between them mid-conversation or let pipelines invoke them automatically.

RAM-aware warnings keep you from overloading your system. You choose how much hardware to commit.

5 model roles RAM monitoring Hot-swap

Pause & Resume

Summarizing a 200-page document? You don't have to sit and wait. NativeLab can pause mid-job, save the exact state — which chunks have been processed, what's been generated — and let you pick it up exactly where you left off, even after restarting.

State serialization Auto-pause threshold Persistent job queue

Reference Library

Build a persistent knowledge base for any chat session. Attach PDFs, notes, or text files as references — they're indexed once and retrieved semantically every time you ask something relevant.

Think of it as giving your AI a bookshelf it can always reach for, separate from the active conversation.

Semantic retrieval Persistent index Per-session libraries

Sessions & Exports

Every conversation is a session with its own history, title, and context window usage indicator. Export to JSON for programmatic use, Markdown for notes and documentation, or plain text for anything else.

Nothing is locked in. Your conversations are yours.

Getting started

Up and running
in four steps

No Docker, no Python environment management, no cloud account. Just download, add a model, and start.

Download NativeLab

Grab the latest Beta release from GitHub. Available for Windows and Linux. Comes bundled with everything it needs.

Add a GGUF model

Drop any .gguf model file into the /localllm folder. NativeLab detects the model family and configures prompt templates automatically.

Configure to your hardware

Set threads, context window size, and generation length in the Config tab. The UI shows RAM usage and context fill in real time so you know you're not overloading anything.

Start working

Open a chat session, attach documents, assign model roles, and start your conversation. Export anytime. Everything persists between sessions.

Model compatibility

Works with the models
you already know

NativeLab automatically detects model families from filenames and applies the correct prompt template. No manual configuration needed for supported families.

Any GGUF-format model that runs on llama.cpp will work. The list below shows families with automatic template detection.

DeepSeek Mistral LLaMA 2 / 3 Qwen / ChatML Phi Gemma CodeLlama Falcon Vicuna OpenChat Yi Command-R Starling Neural-Chat + more

Quantization formats

NativeLab supports the full range of GGUF quantization levels, so you can balance quality and speed for your hardware.

Q2_K → Q8_0 IQ2 / IQ3 / IQ4 Q4_K_M Q5_K_M F16 / F32 BF16

Not sure which quant to pick? Q4_K_M or Q5_K_M strike the best balance for most desktop hardware.

System requirements

What you
actually need

No GPU required. NativeLab runs on CPU with standard RAM — though more RAM gives you larger models and longer contexts.

🖥️ Operating System

  • Windows 10 / 11
  • Linux (most distros)
  • macOS support planned

🧠 Memory (RAM)

  • 8 GB minimum (small models)
  • 16 GB recommended
  • 32 GB for larger models

RAM usage shown live in-app

💾 Storage

  • ~200 MB for the app
  • 2–8 GB per model (varies by quant)
  • Space for sessions & references

⚡ Processor

  • Any modern multi-core CPU
  • More threads = faster generation
  • GPU acceleration: planned

12 threads default, fully configurable
