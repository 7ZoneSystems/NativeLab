Why NativeLab

What you get here
that you don't elsewhere

There are plenty of local-LLM apps. NativeLab differs on a handful of capabilities that the others either don't have, charge for, or hide behind a closed source. Here's the honest list - see the full comparison matrix for line-by-line detail.

🏗️

Visual pipeline builder

Drag-and-drop canvas with 20+ block types - Input, Model, Reference, Intermediate, MERGE, SPLIT, IF/ELSE, LLM-IF, LLM-SCORE, Custom Code. Loops with iteration counts. Live token streaming per stage.

No equivalent in LM Studio · Jan · GPT4All
🧠

LLM-logic blocks

Five blocks where the model itself routes the pipeline - LLM-IF, LLM-SWITCH, LLM-FILTER, LLM-TRANSFORM, LLM-SCORE. Build agentic flows visually, no code.

Exclusive to NativeLab
🧪

Labs - drop-in panels

Add a new feature by dropping one Python file into nativelab/labs/. Every panel gets the same endpoint API for engine status, model swap, context change, and LLM calls.

Exclusive to NativeLab
⌨️

Terminal CLI, same engine

nativelab --cli drops you into a Claude-Code-style chat REPL with @file embedding, slash commands, and built-in linting. Shares the engine layer with the GUI.

No CLI in LM Studio · Jan · GPT4All
⏯️

Pause/resume long jobs

Summarising a 200-page PDF? Full state is serialised every few chunks. Restart the app, hit resume, keep going.

Exclusive to NativeLab
🔓

Fully open source

AGPL v3, every parameter exposed in the UI, codebase developed in the open. LM Studio is proprietary. Read it, fork it, audit it.

20+ block types · 5 LLM-logic blocks · save/load · loop-aware
Draw connections by hand

Drag from any port on a block to any port on another. Smooth Bézier curves. Delete with a click. Whiteboard, not settings panel.

Loops detected automatically

Connect a block back to an earlier one and NativeLab detects the cycle. Set how many times to repeat - it accumulates context each pass.

Every stage streams live

Each Intermediate block gets its own live tab. You see every model's tokens arriving in real time. No black boxes.

Run from chat

Hit the 🔗 Pipeline button next to Send. Pick a saved pipeline and it runs on your current message - each stage appears as its own labelled bubble.

Install & run

One command,
any platform

NativeLab ships as a pure Python package on PyPI. Install with pip - exactly the same on Linux, macOS, and Windows. Requires Python 3.10 or newer.

▸  LINUX · MACOS · WINDOWS
$ pip install nativelab
$ nativelab # desktop GUI
$ nativelab --cli # terminal client

First-time setup wizard

On first launch, the GUI's Download tab installs llama.cpp binaries and pulls a GGUF model - no manual setup. The CLI runs an interactive wizard that finds llama-server, downloads a model, and drops you into a chat REPL.

Build from source

Clone the repository and install in editable mode if you'd like to hack on it.

git clone github.com/7ZoneSystems/NativeLab
cd NativeLab && pip install -e .
