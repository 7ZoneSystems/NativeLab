v0.3.2

1) Fixed error surface in labs endpoints to show real endpoint errors instead of only HTTP 400 text.
2) Added py-to-doc project pause/resume checkpoints in localllm/temp for restart and shutdown recovery.
3) Moved Labs, Logs, Integrations, Pipeline, and MCP under a Dev tab with a contextual left sidebar, and moved Config to the top-right settings button.
4) Added Dev > Skills with a shared localllm skill library, /skills endpoint, and chat toggle for skill injection across chat, labs, and integrations.
5) Added built-in edit skill and structured-edit lab for temp-backed iterative code edits before saving to source files.
6) Added live structured-edit response capture and colored + / - line counts for each model edit.
7) Upgraded structured-edit into a chat-style edit workspace with saved checkpoints, branch-from-history, editable responses, clickable per-turn diffs, live file preview, and Python diagnostics.
8) Added Python diagnostics in Code Edit with syntax checking, lightweight warnings, a Problems panel, and editor line highlighting.
9) Moved detailed Code Edit diffs into per-message chat cards so users can inspect changes for every prompt in the conversation.
10) Fixed Dev tab and Dev sub-tab heading colors so they update correctly when switching dark/light themes.
11) Added independent scrolling to the Code Edit file sidebar so diagnostics, structure, logs, and the editor do not overlap on small screens.
12) Upgraded `nativelab --cli` into a terminal control center with Chat, Models, API Models, Skills, Labs, Pipelines, Integrations, Status, Setup, and Quit.
13) Added shared CLI runtime wiring so chat, Labs, pipelines, endpoint serving, integrations, local GGUF models, API models, and skill injection use the same backend surface.
14) Added CLI commands for models, api-models, skills, labs, pipeline, integrations, endpoint, and serve, with `--json` on inspect/list/show flows.
15) Expanded CLI chat slash commands with `/models`, `/api-models`, `/skills`, `/pipelines`, `/pipeline`, `/labs`, `/py-to-doc`, `/code-edit`, `/endpoint`, and `/serve`.
16) Added CLI model/API management for listing, loading, and setting defaults in `localllm/cli_prefs.json`.
17) Added CLI skill create/edit/delete/enable/disable controls and a saved chat skill toggle.
18) Added CLI Labs support for py-to-doc single/queue/project modes and temp-backed structured code-edit sessions with terminal diffs, save/save-as, and optional editable model JSON responses.
19) Added CLI saved pipeline list/show/run support and foreground integration endpoint serving.
20) Added CLI Discord/WhatsApp profile list/show/create/edit/delete/run commands with redacted secret display.
21) Refreshed CLI docs, beginner guide, README CLI section, and CLI smoke paths for clone/package usage.
22) Enhanced py-to-doc project mode with recursive .gitignore-aware Python discovery and pre-created mirrored output directory structure.
23) Fixed py-to-doc class method discovery to document every direct class method, repair incomplete completed checkpoints, and added explicit project resume controls in GUI and CLI.
24) Expanded explicit server, model-loading, inference, labs, pipeline, download, subprocess, and integration bot timeouts to 24 hours.
25) Added py-to-doc context policies with no-reset, fixed-reset, and auto budget reset modes plus GUI and CLI controls.
26) Synced py-to-doc auto context budgets with local llama.cpp reloads and raised app context caps to 300k tokens.
27) Added a py-to-doc saved project task card that lists checkpoints and resumes selected jobs automatically.
28) Added endpoint-level context-exceeded errors and py-to-doc same-section retry after clearing/reloading context.
29) Added Ollama backend refs (`ollama:<model>`) behind the shared local engine so chat, labs, pipelines, integrations, CLI, and status widgets can use running Ollama models.
30) Added optional Hugging Face Transformers backend refs (`hf:<repo-or-dir>`) with safetensors/local-dir support and image-text model routing where Transformers supports the architecture.
31) Added Models-tab controls for registering Ollama and HF models, plus CLI/backend ref loading for `ollama:` and `hf:` targets.
32) Added Download-tab HF Transformers snapshot downloads with repo inspection, revision support, preserved subdirectories, `.part` resume, pause/cancel controls, token auth, and auto-registration as `hf:<local-folder>`.
33) Added Download-tab Ollama model pulls through an already-running Ollama daemon with installed-model listing, streamed `/api/pull` progress, configurable host, and auto-registration as `ollama:<model>`.
34) Added editable HF Transformers and Ollama backend settings in App Configuration, including HF cache/download dir, token, revision, trust remote code, local-files-only, safetensors, dtype, device map, low-CPU-memory load, attention implementation, max-memory map, quantization, Ollama host, and Ollama keep-alive.
35) Wired `LlamaEngine` HF/Ollama loading and generation to read saved backend settings so chat, Labs, pipelines, integrations, CLI-visible status, and model-role loading share the same runtime behavior.
36) Added curated popular model presets in `templates.py` for GGUF repos, HF Transformers snapshots, and Ollama library models, and wired them into the Download tab as repo/model pickers.
37) Added a top-level Accounts tab with a Hugging Face sub-tab for one-click browser/device OAuth using NativeLab's public client ID, local credential storage in `localllm/cred/huggingface.json`, saved-token reuse for GGUF/HF downloads and `hf:` model loading, an advanced manual-token fallback, masked auth display, and clearer 401/403 login/access errors.
38) Improved gated Hugging Face repo failures so signed-in 403 responses explain that model-specific access may still need acceptance or approval instead of incorrectly asking the user to sign in again.
39) Added py-to-doc auto-mode RAM-triggered local model reloads between files and endpoint-only model reload/wait routing for Labs.
40) Improved Ollama connection handling across downloads, model loading, and chat so connection-refused errors explain that the Ollama daemon must be started or the host setting updated.
41) Reworked Settings into a sidebar with General, Docs, Hugging Face, Ollama, Server, Appearance, and Accounts pages, and hid Dev behind a Developer Mode toggle.
