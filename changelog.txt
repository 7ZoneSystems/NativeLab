v0.3.5
1) Diagnosed the py-to-doc crash path around manual model reloads and local llama.cpp server lifecycle during documentation generation.
2) Added smarter py-to-doc auto-reload routing through LabEndpoints so model/context reload requests go through the host app instead of each lab touching engines directly.

12) Centralized saved model profiles so loading the same model from Chat, Models, CLI, Labs, integrations, or API-server paths reuses the profile saved in Models.
13) Wired saved sampler/profile settings through local llama.cpp server calls, llama-cli calls, Ollama calls, and HF Transformers generation.
14) Added Settings > Accounts data export for full NativeLab app data from `localllm` as one portable file selected through a save dialog.

34) Made the Labs package lazy so importing LabEndpoints in CLI runtime does not import GUI lab panels.
35) Added a defensive early CLI route in `nativelab/main.py` so stale generated wrappers that still import `nativelab.main` route to CLI before GUI imports.
