Two clients, one engine

Run NativeLab as a polished PyQt6 desktop app or as a terminal client with nativelab --cli. Both share the same engine layer, so a model loaded in one can be used from the other.

The CLI runs an interactive setup wizard the first time, downloads a model from HuggingFace, and drops you into a chat REPL with @file embedding, slash commands, and built-in linting via pyflakes / flake8 / pylint.

PyQt6 GUI nativelab --cli Slash commands @file embedding

Labs - the experimentation layer

The nativelab/labs/ package is a sandbox for new features. Every lab panel receives a single LabEndpoints instance and uses it for all engine interaction - status reads, model swap, context change, and synchronous LLM calls (auto-routing API → server → CLI).

Add a feature by dropping nativelab/labs/<feature>.py with a QWidget panel and registering it. The same endpoint surface powers the CLI, so anything you build in Labs works from the terminal too.

LabEndpoints API Drop-in panels Reverse routing

Visual Pipeline Builder

A node-based editor with 20+ block types: I/O (Input, Output, Model, Intermediate), Context (Reference, Knowledge, PDF), Logic (IF/ELSE, SWITCH, FILTER, TRANSFORM, MERGE, SPLIT, Custom Code), and LLM-logic blocks (LLM-IF, LLM-SWITCH, LLM-FILTER, LLM-TRANSFORM, LLM-SCORE).

Every connection is a smooth curve you draw yourself. Build loops with iteration counts, watch each stage stream live in its own tab, save pipelines as JSON, and reload them with one click. Run saved pipelines directly from chat.

API + local mixing

Connect any OpenAI-compatible (/chat/completions) or Anthropic (/v1/messages) endpoint from the API Models tab - works with hosted services and self-hosted servers like LM Studio, Ollama, and vLLM. Paste your key, click Test & Load, and a 1-token check confirms it works before any real request is sent.

Once loaded, an API model behaves exactly like a local one - it appears in chat, works inside pipelines, and shows in the status bar. Keys are stored only on your machine.

OpenAI-compatible Anthropic /v1/messages 1-token verification Local keys only

Talk to your documents

Attach PDFs, plain text, or source files to a session. Top-K relevant chunks are auto-injected as [REFERENCE: …] blocks before each prompt. Long PDFs get split, summarised section-by-section with carry-over context, then consolidated - pause/resume to disk between chunks.

Source code gets AST-level parsing for Python and regex parsers for 20+ other languages, extracting imports, classes, functions, constants, and types.

Top-K RAG Multi-PDF synthesis AST script parser

Auto family detection

NativeLab recognises 20+ chat templates from the filename - DeepSeek, DeepSeek-R1, Mistral, Mixtral, LLaMA-2/3, Phi-3/3.5, Qwen/ChatML, Gemma, CodeLlama, Falcon, Vicuna, OpenChat, Neural-Chat, Starling, Yi, Zephyr, Solar, Orca, Command-R - and applies the correct prompt format automatically.

Every quantization is detected too, from legacy Q4_0 through K-quants and imatrix variants, with colour-coded quality tiers in the model list.

20+ families Quant detection Per-model params

HuggingFace downloader

Search any GGUF repo, browse files with sizes, and download into your models folder - without leaving the app. Network drops? Resume where you left off.

The Download tab also handles HF Transformers snapshots with revision/token support and Ollama pulls through a running daemon. Completed models auto-register as hf:, ollama:, or local GGUF targets.

Search any repo Resume on drop HF snapshots Ollama pulls

Py-to-Doc recovery

Project mode writes checkpoint state after file, class, and function sections. On startup or task selection, NativeLab repairs interrupted project state: running jobs become paused/recovered jobs, incomplete completed checkpoints are reopened, and output offsets are checked before resume.

Auto model reloads are now boundary-aware. When free RAM drops below your configured threshold, the current section finishes first; then NativeLab can unload/reload the active local model before the next LLM section, with cooldowns and re-arm rules to avoid reload loops.

Abrupt failure recovery Paused resume state RAM-aware reloads GUI & CLI

Ollama and HF backends

NativeLab can register ollama:<model> and hf:<repo-or-dir> references next to GGUF files. The same engine surface exposes them to chat, Labs, pipelines, integrations, CLI tools, status widgets, and model-role loading.

HF Transformers settings include local/offline mode, safetensors, dtype, device map, attention implementation, max memory, quantization, trust remote code, revision, cache/download paths, and Hugging Face token reuse.

ollama: refs hf: refs safetensors shared runtime

🎨  Custom prompt formats

Some models need a specific way of wrapping messages. Pick a preset and the fields fill in automatically - or define your own system prompt, user prefix/suffix, and assistant prefix from scratch.

Default ChatML Llama-2 Alpaca Gemma Phi-3 Custom

Adding an API model takes about 30 seconds and it's always tested with one message before anything real is sent - so you know it works before you rely on it.

1
Pick a provider and model

Choose from the list or type any model ID. The base URL fills in automatically for known providers.

2
Paste your API key

Keys are stored only on your machine in a local config file. Nothing is sent anywhere except to the provider you chose.

3
Click Test & Load

NativeLab sends a single 1-token "hi" to verify the connection. Green means ready. Errors appear immediately with the reason so you know exactly what to fix.

4
Use it like any other model

It appears in chat, inside pipelines, and in the status bar. Save the config to reload it instantly next time - no re-entering keys.

Quantization formats

NativeLab supports the full range of GGUF quantization levels, so you can balance quality and speed for your hardware.

Q2_K → Q8_0 IQ2 / IQ3 / IQ4 Q4_K_M Q5_K_M F16 / F32 BF16

Not sure which quant to pick? Q4_K_M or Q5_K_M strike the best balance for most desktop hardware.

System requirements

What you
actually need

No GPU required. NativeLab runs on CPU with standard RAM - though more RAM gives you larger models and longer contexts. API models use no local RAM at all.

🐍 Runtime

  • Python 3.10 or newer
  • PyQt6 (auto-installed)
  • llama.cpp binaries

GUI's Download tab fetches llama.cpp for you

🖥️ Operating System

  • Linux (most distros)
  • macOS (Intel & Apple Silicon)
  • Windows 10 / 11

Same pip install on every platform

🧠 Memory (RAM)

  • 8 GB minimum (small models)
  • 16 GB recommended
  • 32 GB for larger models

RAM shown live in-app · API models use zero local RAM

💾 Storage

  • ~200 MB for the package
  • 2–8 GB per GGUF model
  • Space for sessions, pipelines & references

⚡ Processor

  • Any modern multi-core CPU
  • More threads = faster generation
  • GPU offload via llama.cpp ngl

Threads & GPU layers fully configurable

🌐 Internet

  • Not needed for local models
  • Required only for API model calls
  • No telemetry ever sent

Fully offline with local GGUF models
